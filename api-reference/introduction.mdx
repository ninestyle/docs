The NineStyle REST API gives your application direct access to the full NineStyle platform — browse and manage wardrobe items, assemble and retrieve outfits, pull AI-powered style recommendations, and read or update user profiles. The API follows standard REST conventions: resources are addressed by URL, actions are expressed through HTTP methods, and all payloads are JSON. Whether you're building a mobile wardrobe companion, a personal shopping integration, or an internal tooling dashboard, every capability available in the NineStyle web app is available here.

## Base URL

All API requests must be sent to the following base URL:

```
https://api.ninestyle.com/v1
```

Every request **must** use HTTPS. Requests made over plain HTTP will be rejected with a connection error. The API does not redirect HTTP to HTTPS — always construct URLs with the `https://` scheme.

## Request Format

Send all request bodies as JSON and include the `Content-Type: application/json` header on every request that has a body (`POST`, `PUT`, `PATCH`). Query parameters are used for filtering, pagination, and lightweight options on `GET` requests.

<CodeGroup>
```bash curl
curl --request GET \
  --url https://api.ninestyle.com/v1/wardrobe/items \
  --header "Authorization: Bearer YOUR_API_KEY" \
  --header "Content-Type: application/json"
```

```javascript JavaScript (fetch)
const response = await fetch("https://api.ninestyle.com/v1/wardrobe/items", {
  method: "GET",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
});

const data = await response.json();
console.log(data);
```
</CodeGroup>

## Response Format

Every API response returns JSON. Successful responses wrap the primary payload in a `data` key. List responses additionally include a `meta` key containing pagination information.

**Success shape:**
```json
{
  "data": {
    "id": "item_01HXYZ123456",
    "name": "Classic White Oxford",
    "category": "tops",
    "color": "white",
    "brand": "Everlane",
    "season": "all",
    "image_url": "https://cdn.ninestyle.com/items/item_01HXYZ123456.jpg",
    "created_at": "2024-03-15T09:22:11Z"
  },
  "meta": {
    "request_id": "req_9kJmP2qRsTuV"
  }
}
```

**Error shape:**

When a request fails, the API returns an `error` object with a machine-readable `code` and a human-readable `message`. There is never a `data` key on an error response.

```json
{
  "error": {
    "code": "resource_not_found",
    "message": "No wardrobe item with ID item_01HXYZ999999 was found for this account."
  }
}
```

## Rate Limits

The API enforces per-minute rate limits based on your subscription plan:

| Plan | Limit |
|------|-------|
| Free | 60 requests / minute |
| Pro  | 600 requests / minute |

Every response includes the following headers so your application can track consumption:

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | The total number of requests allowed in the current window |
| `X-RateLimit-Remaining` | The number of requests remaining in the current window |
| `X-RateLimit-Reset` | Unix timestamp (UTC) at which the window resets |

When you exceed your limit the API returns **HTTP 429 Too Many Requests**. Back off and retry after the timestamp in `X-RateLimit-Reset`.

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "You have exceeded your plan's rate limit of 60 requests per minute. Retry after 1716211260."
  }
}
```

## Pagination

All list endpoints accept the following query parameters:

| Parameter | Type | Default | Max | Description |
|-----------|------|---------|-----|-------------|
| `page` | integer | `1` | — | The page number to retrieve |
| `per_page` | integer | `20` | `100` | Number of results per page |

Every list response includes a `meta.pagination` object:

```json
{
  "data": [ "..." ],
  "meta": {
    "pagination": {
      "total": 143,
      "per_page": 20,
      "current_page": 2,
      "total_pages": 8,
      "next_page": 3,
      "prev_page": 1
    }
  }
}
```

`next_page` and `prev_page` are `null` when there is no following or preceding page respectively.

</Note>
