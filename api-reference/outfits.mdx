sidebarTitle: "Outfits"
description: "REST endpoints to list, create, update, and delete outfits. An outfit is a named collection of wardrobe item IDs assembled into a wearable look."
---

An outfit is a curated grouping of wardrobe items that together form a complete look — for example, a white Oxford shirt, navy chinos, and tan leather loafers assembled for a business-casual day. Each outfit object stores the IDs of its constituent wardrobe items, an optional cover photo, a 1–5 star rating, an occasion tag, and a visibility flag that controls whether the outfit appears in NineStyle's public style feed. Use the Outfits API to let users save looks they love, build outfit calendars, share styles publicly, and feed structured data to the recommendations engine.

---

## GET /outfits

Retrieve a paginated list of outfits belonging to the authenticated user. Use the `public` parameter to filter between public and private looks.

**Required scope:** `outfits:read`

### Query Parameters

<ParamField query="page" type="integer" default="1">
  The page number to retrieve.
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Number of outfits to return per page. Maximum `100`.
</ParamField>

<ParamField query="public" type="boolean">
  When `true`, return only outfits the user has marked as public. When `false`, return only private outfits. Omit to return all outfits regardless of visibility.
</ParamField>

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the outfit, prefixed `outfit_`.
</ResponseField>

<ResponseField name="name" type="string">
  Display name of the outfit, e.g. `"Summer Rooftop Look"`.
</ResponseField>

<ResponseField name="items" type="array">
  Array of wardrobe item IDs included in the outfit. Use `GET /outfits/:id` to retrieve the full item objects.
</ResponseField>

<ResponseField name="image_url" type="string">
  Cover photo URL for the outfit hosted on the NineStyle CDN. `null` if no image has been set.
</ResponseField>

<ResponseField name="occasion" type="string">
  Occasion tag for the outfit, e.g. `"casual"`, `"work"`, `"evening"`. `null` if not set.
</ResponseField>

<ResponseField name="rating" type="integer">
  User's self-rated score from `1` to `5`. `null` if not rated.
</ResponseField>

<ResponseField name="public" type="boolean">
  Whether this outfit is visible on the NineStyle public style feed.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the outfit was created.
</ResponseField>

### Examples

<CodeGroup>
```bash curl
curl --request GET \
  --url "https://api.ninestyle.com/v1/outfits?public=false&per_page=10" \
  --header "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript (fetch)
const params = new URLSearchParams({ public: "false", per_page: "10" });

const response = await fetch(

  {
```

## POST /outfits

Create a new outfit by naming it and providing an array of wardrobe item IDs. All items must belong to the authenticated user.

**Required scope:** `outfits:write`

### Body Parameters

<ParamField body="name" type="string" required>
  Display name for the outfit, e.g. `"Friday Night Dinner"`. Maximum 120 characters.
</ParamField>

<ParamField body="items" type="array" required>
  An array of wardrobe item ID strings to include in the outfit. Must contain at least one ID. All IDs must belong to the authenticated user's wardrobe.
</ParamField>

<ParamField body="occasion" type="string">
  Optional occasion tag for the outfit. Common values: `casual`, `work`, `evening`, `sport`, `formal`, `travel`.
</ParamField>

<ParamField body="rating" type="integer">
  Optional self-rated score from `1` (lowest) to `5` (highest).
</ParamField>

<ParamField body="public" type="boolean" default="false">
  Set to `true` to publish the outfit to the NineStyle public style feed.
</ParamField>

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the newly created outfit, prefixed `outfit_`.
</ResponseField>

<ResponseField name="name" type="string">
  The display name supplied in the request body.
</ResponseField>

<ResponseField name="items" type="array">
  Array of wardrobe item ID strings included in the outfit, matching the IDs supplied in the request.
</ResponseField>

<ResponseField name="image_url" type="string">
  Cover photo URL for the outfit. `null` until an image is explicitly set via a subsequent update.
</ResponseField>

<ResponseField name="occasion" type="string">
  Occasion tag supplied in the request body. `null` if not provided.
</ResponseField>

<ResponseField name="rating" type="integer">
  Self-rated score from `1` to `5` supplied in the request body. `null` if not provided.
</ResponseField>

<ResponseField name="public" type="boolean">
  Whether the outfit is visible on the NineStyle public style feed. Reflects the value supplied in the request, defaulting to `false`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the outfit was created.
</ResponseField>

<CodeGroup>
```bash curl
curl --request POST \
  --url https://api.ninestyle.com/v1/outfits \
  --header "Authorization: Bearer YOUR_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "name": "Friday Night Dinner",
    "items": ["item_01HXYZ111111", "item_01HXYZ333333", "item_01HXYZ555555"],
    "occasion": "evening",
    "rating": 5,
    "public": false
  }'
```

```javascript JavaScript (fetch)
const response = await fetch("https://api.ninestyle.com/v1/outfits", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Friday Night Dinner",
    items: ["item_01HXYZ111111", "item_01HXYZ333333", "item_01HXYZ555555"],
    occasion: "evening",
    rating: 5,
    public: false,
  }),
});

const { data } = await response.json();
console.log(data.id); // "outfit_01HABC333333"
```

## DELETE /outfits/:id

Permanently delete a saved outfit. The wardrobe items that were part of the outfit are **not** deleted — only the outfit grouping itself is removed.

**Required scope:** `outfits:write`

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the outfit to delete.
</ParamField>

<Warning>
  Deleting an outfit is **permanent and irreversible**. The outfit and any
  associated cover photo are removed immediately. The individual wardrobe items
  inside the outfit are unaffected and remain in the user's wardrobe.
</Warning>

A successful deletion returns **HTTP 204 No Content** with an empty response body.

<CodeGroup>
```bash curl
curl --request DELETE \
  --url https://api.ninestyle.com/v1/outfits/outfit_01HABC333333 \
  --header "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript (fetch)
const response = await fetch(
  "https://api.ninestyle.com/v1/outfits/outfit_01HABC333333",
  {
    method: "DELETE",
    headers: { "Authorization": "Bearer YOUR_API_KEY" },
  }
);

if (response.status === 204) {
  console.log("Outfit deleted successfully.");
}
```
</CodeGroup>
