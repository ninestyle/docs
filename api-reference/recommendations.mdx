The Recommendations API exposes NineStyle's AI styling engine directly to your application. Each time you call `GET /recommendations`, the engine analyses the authenticated user's wardrobe inventory, their saved style preferences (preferred aesthetics, color palette, and budget range), live weather data for their location, and any calendar events connected to their account. It then assembles ranked outfit suggestions or personalised shopping prompts and returns them with plain-language reasoning so users understand why each suggestion was made. Submit feedback after a recommendation is shown to continuously refine the model for that user.

---

## GET /recommendations

Fetch a list of AI-generated style recommendations for the authenticated user.

**Required scope:** `recommendations:read`

### Query Parameters

<ParamField query="type" type="string" default="outfit">
  The type of recommendation to generate. Use `outfit` to receive curated looks assembled from existing wardrobe items, or `shopping` to receive suggestions for new items that would complement the user's current wardrobe.
</ParamField>

<ParamField query="occasion" type="string">
  Constrain recommendations to a specific occasion. Common values: `casual`, `work`, `evening`, `sport`, `formal`, `travel`. Omit to let the engine choose the best occasion based on the user's calendar.
</ParamField>

<ParamField query="limit" type="integer" default="5">
  Number of recommendations to return. Minimum `1`, maximum `20`.
</ParamField>

<ParamField query="weather_aware" type="boolean" default="true">
  When `true`, the engine filters out clothing unsuitable for the user's current local weather conditions (e.g. heavy coats on a warm day). Set to `false` to disable weather filtering.
</ParamField>

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the recommendation, prefixed `rec_`. Use this ID when submitting feedback.
</ResponseField>

<ResponseField name="type" type="string">
  The recommendation type: `outfit` or `shopping`.
</ResponseField>

<ResponseField name="outfit" type="object">
  The recommended outfit or shopping suggestion.

  <Expandable title="outfit fields">
    <ResponseField name="outfit.name" type="string">
      A generated display name for the look, e.g. `"Breezy Weekend Brunch"`.
    </ResponseField>
    <ResponseField name="outfit.items" type="array">
      Array of objects, each containing the `id` and `name` of a wardrobe item (for `outfit` type) or a suggested purchase (for `shopping` type).
    </ResponseField>
    <ResponseField name="outfit.image_url" type="string">
      AI-generated or collaged preview image URL for the recommended look. `null` if not available.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="reasoning" type="string">
  A plain-language explanation of why the engine surfaced this recommendation, e.g. `"You have a dinner reservation tonight and the forecast is 18 °C with clear skies — this linen look keeps you cool while looking sharp."`.
</ResponseField>

<ResponseField name="score" type="float">
  A confidence score between `0.0` and `1.0` indicating how well this recommendation fits the user's style profile. Higher is better.
</ResponseField>

### Examples

<CodeGroup>
```bash curl
curl --request GET \
  --url "https://api.ninestyle.com/v1/recommendations?type=outfit&occasion=work&limit=3" \
  --header "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript (fetch)
const params = new URLSearchParams({
  type: "outfit",
  occasion: "work",
  limit: "3",
});

const response = await fetch(
  `https://api.ninestyle.com/v1/recommendations?${params}`,
  {
    headers: { "Authorization": "Bearer YOUR_API_KEY" },
  }
);

const { data } = await response.json();
data.forEach((rec) => {
  console.log(`${rec.outfit.name} — score: ${rec.score}`);
  console.log(rec.reasoning);
});
```
</CodeGroup>

## POST /recommendations/:id/feedback

Submit a like, dislike, or neutral rating on a recommendation to help NineStyle's AI tailor future suggestions for this user. The engine incorporates feedback signals in real time — a disliked recommendation style will appear less frequently in subsequent calls.

**Required scope:** `recommendations:read`

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the recommendation to rate, e.g. `rec_01HREC111111`.
</ParamField>

### Body Parameters

<ParamField body="rating" type="string" required>
  The user's reaction to the recommendation. Must be one of:
  - `"like"` — the user loved the suggestion
  - `"dislike"` — the user didn't want to see this type of suggestion
  - `"neutral"` — acknowledged but no strong preference
</ParamField>

<CodeGroup>
```bash curl
curl --request POST \
  --url https://api.ninestyle.com/v1/recommendations/rec_01HREC111111/feedback \
  --header "Authorization: Bearer YOUR_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{ "rating": "like" }'
```

```javascript JavaScript (fetch)
const response = await fetch(
  "https://api.ninestyle.com/v1/recommendations/rec_01HREC111111/feedback",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer YOUR_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ rating: "like" }),
  }
);

const { data } = await response.json();
console.log(data.rating); // "like"
```
</CodeGroup>

```json Example Response (200 OK)
{
  "data": {
    "id": "rec_01HREC111111",
    "rating": "like",
    "feedback_recorded_at": "2024-06-18T20:45:00Z"
  },
  "meta": {
    "request_id": "req_6fFgH9iJkL10"
  }
}
```

<Note>
  Every feedback signal you submit directly improves the quality of future
  recommendations for this user. Encourage your users to rate suggestions in
  your UI — even a simple thumbs up/down interaction makes a measurable
  difference in personalisation accuracy over time.
</Note>
